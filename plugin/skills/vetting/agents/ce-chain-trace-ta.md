## Check — TA cost denominator consistency

A TA BOTEC commonly defines two different cost figures:
- **Grant amount**: funding requested for this grant period only (e.g., $4.9M for 1.5 years)
- **Total cost of TA engagement**: full cost through TA exit across all grant periods (e.g., $11.4M)

Both are defensible denominators. Using total engagement cost amortizes benefits across the full TA investment; using grant amount evaluates only this grant's marginal cost. Either is acceptable if used consistently and documented. The error pattern is **inconsistency between tabs**: the Main CEA uses one cost base while the Simple CEA uses another, making their CE multiples non-comparable without the reader knowing why they differ.

First, verify the Simple CEA tab exists in the workbook (check session context tab list). When searching for the Simple CEA tab, use flexible matching — accept any of the following as a match: exact name "Simple CEA", "Simple CE", "Simple Model", "Simplified CEA", or any tab whose name contains "Simple" and at least one of "CEA", "CE", or "Model" (case-insensitive). If multiple tabs match, use the first match in tab order. Log the detected tab name in your AGENT_COMPLETE marker (e.g., `simple-cea-tab-detected: "Simplified CEA"`). If no matching tab is present: file a Low/Assumption finding: "Simple CEA tab absent — cannot verify cost denominator consistency." Then write your AGENT_COMPLETE marker and stop. Do not attempt to read a non-existent tab.

After confirming TA signals, read the TA Modeling Guidance document (use `get_doc_content` on ID `12onXe086vgvSBSVCbIwvWwWft7JYIpPpJ--CK80Ez6M` (TA Modeling Guidance; also in the reference table as doc #9 and in session context when is_ta_botec: true)) to determine whether GW TA convention recommends a specific cost denominator.
