Spreadsheet Vetting Skill — User Guide

From installation to reading your output

1 · Setup
1
One-time
Install the Hardened Google Workspace MCP

The skill needs this MCP to read your Google Sheet and write results. Set it up once; it persists across sessions.

Contact Nicole Bouchard (nicole.bouchard@givewell.org) for the setup snippet, or follow the instructions at:

github.com/c0webster/hardened-google-workspace-mcp
Can't install the MCP? The skill also runs in local output mode — download your spreadsheet as .xlsx and Claude writes findings to CSV files you import manually. You lose notes/hyperlink checks but all formula and CE chain checks still run.
2
One-time
Install the vetting skill (plugin — recommended)

In Claude Code (CLI, desktop app, or cowork on the web), run:

/plugin marketplace add meghna-givewell/givewell-vetting-skill
/plugin install givewell-vetting@givewell-skills

The plugin auto-updates on startup — you always get the latest agent calibrations without any extra steps.

Alternative — standalone clone: If you prefer a local install, clone the repo into your skills directory:
git clone https://github.com/meghna-givewell/givewell-vetting-skill.git ~/.claude/skills/vetting
With this method, update manually before each vet: git -C ~/.claude/skills/vetting pull --rebase origin main
3
One-time
Copy the permissions settings to your project

Pre-approve all required MCP tool calls so Claude doesn't pause for permission prompts during the vet:

cp ~/.claude/skills/vetting/.claude/settings.json <your-project>/.claude/settings.json

If your project already has a .claude/settings.json, merge the allow array entries from the skill's file into yours.

4
Each vet
Update the skill before running

Agent calibrations are refined after every completed vet. Update first:

  • Plugin install: auto-updates on startup. To pull immediately: /plugin marketplace update givewell-skills
  • Standalone clone: git -C ~/.claude/skills/vetting pull --rebase origin main
The skill checks its own freshness when it starts and will warn you if it detects commits you haven't pulled.
2 · Running the Vet
5
Each vet
Invoke the skill with your spreadsheet URL
# Plugin install
/givewell-vetting:vetting https://docs.google.com/spreadsheets/d/<your-id>/edit

# Standalone clone
/vetting https://docs.google.com/spreadsheets/d/<your-id>/edit

The skill accepts a Google Sheets URL or a local .xlsx file path.

6
Each vet
Answer the startup questions (one combined message)

Claude will ask all of these in a single message — reply to all at once so reads and literature searches can fire in parallel:

Q1 — Email
What is your Google Workspace email address?
Used for MCP authentication. Required at the start of every session.
Q2 — Sheets & scope
Which sheets should I vet, and is this headed toward publication or is it internal/early-stage?
Publication scope = full checks. Internal/early-stage = formula and assumption checks only (sources, readability, citations skipped).
Q3 — Grant context
Is there a grant description, one-pager, or write-up I should read first?
Optional. If provided, Claude fetches and reads it in the same parallel batch.
Q4 — Intentional deviations
Have you deliberately set any parameters differently from GiveWell defaults? If so, list them.
Declared deviations are verified and capped at Low severity rather than filed as errors.
Q5 — Prior research
Is there a GiveWell intervention analysis, prior-version CEA, or internal research note for this health area?
Used by plausibility agents to cross-check model inputs against established GW values.
Q6 — Source citation verification
Should I run source citation verification for Study-Derived and Org-Reported hardcoded inputs? [Yes / No]
Optional Wave 1.5. Fetches each cited source and uses the Citations API to confirm values are supported — pre-fills the Verified? column. GiveWell parameter consistency is always checked regardless.
7
Each vet
Confirm the orientation summary before analysis begins

After reading the spreadsheet, Claude produces a brief summary of each sheet's purpose and section structure. Review it and reply to confirm (or correct any misreadings) before the multi-wave analysis starts.

Why this matters: Any misidentification of the CE baseline cell or section structure here will propagate to every subsequent sub-agent. Spend 60 seconds checking this before clicking through.
8
During
Wait for the four-wave analysis to complete

Claude spawns parallel agents across four waves. Typical runtimes vary by sheet complexity (15–50 min for a top-charity CEA). You'll see progress announcements for each wave.

Wave 1
Formula errors, parameter accuracy (stale values, GBD vintages, country mismatches), structural completeness, cross-parameter consistency, VOI checks, confidentiality flags, hardcoded values inventory
Wave 1.5
Optional — source citation verification. Fetches each cited source; fills Verified? and Auto-check evidence columns in the Hardcoded Values sheet
Wave 2
Data sources and citations, CE sanity, epidemiological plausibility, intervention-specific checks, leverage/funging, CE chain trace, readability, notes quality
Wave 2.5
Reconciliation — resolves divergences between independent parallel agent instances covering the same scope
Wave 3
Compaction (deduplicate, sort, assign Finding IDs F-001…), gap-fill, validation, dashboard summary
If interrupted before Wave 3 completes: Run /givewell-vetting:vetting-finalize (plugin) or /vetting-finalize (standalone) in a new session. Provide the output spreadsheet URL and source spreadsheet URL when asked — it recovers from where the session stopped.
3 · The Output Spreadsheet
9
Output
Six tabs — what each one contains
Tab Contains Who uses it
Findings
 Model-integrity issues (formula errors, stale parameters, structural bugs). Sorted High → Medium → Low, with Finding IDs (F-001…). 10 columns. Lead Investigator, CRA — primary review
Publication Readiness
 Issues that don't affect model outputs: missing sources, broken links, Box links, terminology (x cash → x benchmark), permission flags. Finding IDs (PR-001…). 6 columns. Lead Investigator — pre-publication only; can ignore until then
Hardcoded Values
 Inventory of every standalone hardcoded cell. Column G (Verified?) and H (Auto-check evidence) pre-filled by Wave 1.5 when source citation verification is enabled. Researcher — verify each row; mark Yes / No / In Progress in column G
Confidentiality Flags
 Named individuals, donor info, PII, unpublished strategy found in the spreadsheet. Lead Investigator — resolve before any external sharing
CE Baseline
 Pre-vet cost-effectiveness figures by geography/scenario, with cell references. Used as the reference for CE impact estimates throughout Findings. Reviewer context — reference only
Dashboard
 Summary: finding counts by sheet, overall CE direction, list of unvetted tabs, key findings highlighted in chat. Manager / CRA — quick status overview
10
Output
Understanding finding severity

Severity is determined by two factors: Nature (is this objectively wrong, or a judgment call?) and Materiality (what does it do to the CE estimate?).

High
Affects CE by ≥5%, flips the funding decision, or violates a GiveWell standard parameter (benchmark, moral weight) with no documented rationale
Medium
Confirmed error with <5% CE impact; or a required element is absent (missing source, missing adjustment); floor for any confirmed formula error
Low
Defensible modeling choice worth flagging; documentation gap on a value that is itself correct; style / label issues

If CE impact direction or magnitude can't be determined, severity rounds up — so a formula error in the CE chain with unknown impact is High.

11
Output
Understanding error types
Formula
Wrong cell reference, wrong range, broken logic, incorrect operator. Always shows a bracketed sub-type: [Copy-paste], [Wrong reference], [Year range], [Sign error], etc.
Parameter
Hardcoded value is stale, outdated, or conflicts with a GiveWell cross-cutting standard (moral weight, benchmark, discount rate).
Adjustment
An adjustment (IV, EV, leverage, funging) is absent, has the wrong sign, wrong base, or is applied multiplicatively vs. additively incorrectly.
Assumption
Key assumption lacks a source or explanation; structural model issue (missing required tab, inverted section order).
Inconsistency
The same value appears differently in two places, or a cross-sheet reference pulls a value that doesn't match its stated source.
Legibility
Tab or section ordering, unclear or stale labels, placeholder text, terminology errors (e.g., "x cash" → "x benchmark").

On the Publication Readiness tab, error types are simplified to three: Sourcing, Box Link, Legibility.

12
Output
Reading the CE Impact column

Column H in the Findings sheet always uses one of these exact phrases:

  • Raises CE — [estimate] — error makes the program look better than it is; fixing it lowers CE
  • Lowers CE — [estimate] — error makes the program look worse; fixing it raises CE
  • Raises/Lowers CE — magnitude unknown — direction is clear; size can't be computed without researcher input
  • No CE impact — confirmed zero effect on the bottom line
  • Direction unknown — even the direction requires a researcher decision (e.g., a placeholder with no indicated range)
A blank column H means the agent did not assess CE impact — treat these as unresolved. Findings marked Researcher judgment needed ✓ require you to make a modeling decision before the finding can be resolved.
4 · After the Vet
13
Researcher
Work through the Findings sheet — mark each finding's Status

Column J (Status) is yours to fill in. Use one of four values:

Open — not yet reviewed
Fixed — corrected in the spreadsheet
Won't Fix — intentional; note your reasoning in the cell
Needs Discussion — requires a team decision

Prioritize by severity: address all High findings before Medium. For High findings marked Researcher judgment needed ✓, answer the question in column F before deciding.

14
Researcher
Verify the Hardcoded Values sheet

For each row, confirm the value is supported by its cited source. Column G tracks your progress:

  • If Wave 1.5 ran: Claude pre-filled G with Matched ✓, Contradicted ✗, or Could not verify — and column H shows the verbatim sentence from the source. Review and override as needed.
  • If Wave 1.5 did not run: verify manually. Update G to Yes, No, or In Progress.
Values labeled "guess" or "best guess" in cell notes are not errors — this is acceptable uncertainty documentation and does not require a finding.
15
Pre-publication
Resolve Publication Readiness and Confidentiality Flags before sharing externally

The Confidentiality Flags sheet must be cleared (all rows addressed) before any external sharing or publication. The Publication Readiness sheet should be cleared before the spreadsheet goes through formal publication review.

For Box links flagged in Publication Readiness: replace with a public link or remove — Box URLs are not appropriate in externally published spreadsheets.

16
Optional
Re-run after revisions (pre-publication)

After resolving High and Medium findings and revising the spreadsheet, run the skill again on the updated version — this catches regressions introduced during fixes and runs the full Publication Readiness check on the final model.

# Same command, same spreadsheet URL (or updated URL if you saved a copy)
/givewell-vetting:vetting https://docs.google.com/spreadsheets/d/<your-id>/edit
Leave a pilot feedback response at the end of each vet. Claude will ask five short questions. Your answers feed the calibration file that every agent reads before starting — this is how false positives get eliminated and missed findings get caught in future vets.
